# Функциональные требования к виртуальному скроллу

## 1. Назначение

Обеспечить эффективную прокрутку больших наборов данных с поддержкой:

- фиксированных и динамических размеров;
- фиксированных строк и столбцов;
- программного управления скроллом.

---

## 2. Обязательные возможности

### 2.1 Виртуализация строк

- Рендерится только диапазон видимых строк.
- Поддержка overscan (настраиваемый).
- Поддержка fixed-height и dynamic-height.

---

### 2.2 Виртуализация столбцов

- Аналогично строкам:
  - фиксированная ширина;
  - динамическая ширина;

- Возможность включения/отключения отдельно от строк.

---

### 2.3 Фиксированные строки и столбцы

- Верхние/нижние строки (header/footer).
- Левые/правые столбцы.
- Корректная синхронизация со скроллом.
- Не участвуют в виртуализации.

---

### 2.4 Прокрутка к указанной строке / ячейке

```ts
scrollToIndex(index: number, options?: {
  align?: 'start' | 'center' | 'end';
  behavior?: 'auto' | 'smooth';
});
```

```ts
scrollToCell(rowIndex: number, columnIndex: number);
```

**Требования:**

- Работает для fixed и dynamic размеров.
- Поддерживает ещё не измеренные элементы.
- Сохраняет корректный scroll anchor.

---

## 3. Поведение по состояниям

| Состояние    | Поведение            |
| ------------ | -------------------- |
| loading      | placeholder строки   |
| empty        | zero-height viewport |
| error        | внешний контроль     |
| resizing     | пересчёт диапазонов  |
| data prepend | anchoring            |

---

## 4. Ограничения и правила

- Минимальный overscan ≥ 1 элемент.
- Dynamic mode требует estimatedSize.
- Sticky элементы не должны быть частью virtual window.

---

## 5. События и коллбеки

```ts
onRangeChange(range: {
  rows: VirtualRange;
  columns?: VirtualRange;
});
```

```ts
onScroll(position: {
  top: number;
  left: number;
});
```

---

## 6. Нефункциональные требования

### Производительность

- ≤ 100 DOM-нод в среднем viewport.
- Без layout thrashing.
- Scroll handler через `requestAnimationFrame`.

### Доступность

- Не ломать native scroll.
- Совместимость с keyboard navigation (через consumer).

### Совместимость

- Desktop / touch.
- SSR: fallback render первых N элементов.

---

## 7. Анти-сценарии

Виртуальный скролл **не подходит**, если:

- элементов < 100;
- требуется полный DOM (Ctrl+F по странице);
- сложная inline-верстка с авто-высотой без ограничений.

---

## 8. Критерии готовности (DoD)

- ScrollToIndex покрыт тестами.
- Fixed / dynamic режимы работают идентично по API.
- Нет скачков при resize и prepend.
- Документированы trade-offs.
