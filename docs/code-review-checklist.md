# Code Review Checklist — Virtualization Library

## 0) Метаданные PR

* [ ] В PR есть **краткое описание**: что меняется и почему.
* [ ] Указан тип изменений: `feat | fix | refactor | perf | docs | test | chore`.
* [ ] Есть ссылка на issue / ADR / обсуждение (если применимо).
* [ ] Если изменения затрагивают поведение/контракт — указан **migration path** или пояснение.

---

## 1) Соответствие ADR (архитектурные инварианты)

### ADR-001 (Architecture)

* [ ] Логика расчётов (ranges/offsets/strategies) остаётся в **core**, React-слой — только адаптер.
* [ ] 1D (List) не содержит отдельного “особого” алгоритма, который дублирует 2D; используется общая модель осей.

### ADR-002 (Public API)

* [ ] Публичный API не расширен “случайными” экспортами (внутренности типа `AxisModel`, `SizeIndex` не утекли наружу).
* [ ] Изменения публичных типов/props:

  * [ ] обратно совместимы, или
  * [ ] оформлены как breaking change с планом депрекейта.

### ADR-003 (Scroll Anchoring)

* [ ] Изменения не отключают anchoring в dynamic-режиме.
* [ ] Нет прямых коррекций `scrollTop/scrollLeft` без механизма anchoring (кроме явно описанных случаев).

### ADR-004 (Sticky)

* [ ] Sticky-элементы не попали в виртуализируемое окно и не участвуют в AxisModel.
* [ ] Corner-зоны корректно обрабатываются (если есть top+left и т.п.).

### ADR-005 (Dynamic Size Index)

* [ ] В dynamic-режиме нет `O(n)` проходов в hot-path (scroll/resize/range calc).
* [ ] Операции `findIndexByOffset/getOffset/setSize` остаются `O(log n)`.

---

## 2) Публичный API и DX

* [ ] API минимален: добавлены только действительно необходимые props/опции.
* [ ] Названия и семантика опций согласованы:

  * `sizeMode: fixed|dynamic`
  * `estimatedItemSize` обязателен для dynamic
  * `overscan` единообразен (1D/2D)
* [ ] Добавлены/обновлены примеры использования (если затрагивает API).
* [ ] Документация (README/Storybook Docs) обновлена, если пользовательский контракт изменился.

---

## 3) Корректность алгоритмов и математики

* [ ] Range calculation корректно работает на границах:

  * начало списка (scroll=0),
  * конец (scroll ≈ totalSize),
  * маленький viewport,
  * count = 0 / 1.
* [ ] Clamp индексов везде, где возможно out-of-bounds.
* [ ] Для dynamic:

  * [ ] корректно используется `estimatedItemSize` до измерения,
  * [ ] повторные измерения не приводят к дрожанию (лишним апдейтам).
* [ ] `scrollToIndex/scrollToCell`:

  * [ ] корректно поддерживает `align`,
  * [ ] предсказуем в dynamic до измерения,
  * [ ] не ломает anchoring.

---

## 4) Производительность и рендер

### Scroll pipeline

* [ ] Обработчик scroll не делает тяжёлых синхронных вычислений.
* [ ] Изменение состояния на скролле батчится (например, через RAF / throttling).
* [ ] Нет layout thrashing (чтение layout + запись стилей в одной фазе без батчинга).

### DOM и стили

* [ ] Используется позиционирование окна через `transform`/offset без лишних reflow.
* [ ] Количество DOM-нод ограничено viewport + overscan.
* [ ] Sticky-слои не создают неконтролируемого роста DOM.

### Измерения

* [ ] `ResizeObserver`/measurement:

  * [ ] включается только при dynamic,
  * [ ] корректно очищается (unsubscribe),
  * [ ] апдейты размеров агрегируются (без каскада).

---

## 5) Стабильность скролла (UX-инварианты)

* [ ] Нет scroll jump при:

  * загрузке изображений/контента в видимой зоне,
  * expand/collapse элементов,
  * prepend данных.
* [ ] Sticky не “дёргается” при горизонтальном/вертикальном скролле.
* [ ] В быстрых прокрутках нет “пустых дыр” (overscan достаточен или корректно настроен).

---

## 6) Edge cases и отказоустойчивость

* [ ] Корректная работа при:

  * `count=0`, `items=[]`,
  * очень больших индексах scrollTo (clamp),
  * отрицательных scroll offsets (защита),
  * NaN/0 размеров (fallback на estimate).
* [ ] При ошибках измерений не происходит бесконечных циклов setState/measure.

---

## 7) Тесты

### Unit (core)

* [ ] Добавлены/обновлены unit-тесты для:

  * range calc,
  * dynamic size index (prefix sums),
  * findIndexByOffset граничные случаи,
  * anchoring расчёты.

### Integration (React)

* [ ] Есть интеграционные тесты на ключевые сценарии:

  * dynamic + late-measure,
  * prepend без jump,
  * scrollToIndex в dynamic,
  * sticky+virtual body синхронизация.

### Регрессии

* [ ] Исправление багов сопровождается тестом, который падал до фикса.

---

## 8) Типы, совместимость, качество кода

* [ ] TypeScript типы не “размыты” (нет `any` без обоснования).
* [ ] API типы экспортируются только из публичных entrypoints.
* [ ] Нет циклических импортов между слоями (`core` ↔ `react`).
* [ ] Комментарии — только для архитектурных решений; нет “шумных” комментариев.
* [ ] Нет зависимостей, которые тащат большой runtime без необходимости.

---

## 9) Документация и примеры

* [ ] Обновлены docs по поведению:

  * dynamic mode требования (`estimatedItemSize`),
  * anchoring гарантии и ограничения,
  * sticky зоны и corner.
* [ ] Добавлены/обновлены Storybook stories (если затрагивает UI-слой):

  * базовые,
  * состояния (empty/loading),
  * dynamic,
  * sticky,
  * scrollTo demo.

---

## 10) Релизные заметки и миграции (если нужно)

* [ ] Если PR меняет поведение — добавлена запись в CHANGELOG / changeset.
* [ ] Если есть депрекейт — добавлено предупреждение и срок удаления.
* [ ] Описаны риски и план отката (если изменения нетривиальные).

---

## “Stop the line” критерии (блокирующие)

PR **не должен быть принят**, если:

* [ ] есть breaking change без ADR/депрекейта;
* [ ] в dynamic hot-path появился `O(n)` на scroll;
* [ ] anchoring отключён/сломлен;
* [ ] sticky попал в виртуализируемое окно;
* [ ] нет тестов на исправленный баг / ключевой алгоритм.
